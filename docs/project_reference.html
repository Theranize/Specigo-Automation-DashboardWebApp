2 Quick Start

# Run all E2E tests
.\Test\Scripts\pytest.exe tests/e2e -v --tb=short

# Run by marker
.\Test\Scripts\pytest.exe -m e2e -v
.\Test\Scripts\pytest.exe -m smoke -v
.\Test\Scripts\pytest.exe -m regression -v

# Run specific test file
.\Test\Scripts\pytest.exe tests/e2e/acceptance/test_e2e_acceptance.py -v

# Generate Allure HTML report
allure generate reports/allure-results -o reports/allure-report --clean
Always use .\Test\Scripts\pytest.exe (venv), NOT the system pytest.

3 E2E Flow — Role Sequence

StepRoleKey Actions
1Front DeskRegister patient, assign tests, complete payment, print barcode, capture sample IDs
2PhlebotomistSearch patient, toggle samples to "collected"
3AccessionSearch patient, Accept / Reject / Refresh each sample
4Lab TechnicianOpen Report Entry, navigate sub-depts, fill parameters, save each test
5DoctorOpen Report Entry, navigate sub-depts, fill params, perform action

Doctor Actions

ActionSteps
approveFill params → Save → Approve → "Fully Approve" dialog
partial_approveFill params → Save → Approve → "Partially Approve" dialog
retestClick Re-Test → Confirm Yes
resampleClick Resample → select/enter reason → Submit
rectifyFill params → Rectify (no Save) → dialog → reason → optional Others text → Submit → Yes

DDT Lookup Pattern

from utils.file_utils import load_json
from pathlib import Path

DATA = load_json(Path("test_data/role/file.json"))

def _entry(lst, key, val):
    return next(e for e in lst if e[key] == val)

# In test:
entry = _entry(DATA["patients"], "patient_id_ref", "P1")
result = execute_some_flow(page, entry)
assert result["completed"]

14 Key Coding Patterns

AntD Quirks

ProblemSolution
Overlay intercepts clickUse JS dispatch: el.evaluate("el => el.click()")
Combobox with pre-selected itemUse click(force=True)
Salutation includes dot (Mr.)Normalize in select_salutation method
Search comboboxUse type() with delay=80, NOT fill()
#rc_select_N IDs shift dynamicallyAlways use label-based XPath, never ID-based

General Rules

RuleReason
Scroll before below-fold elementsAntD lists require element visibility before interaction
Payment/number inputs → type() not fill()fill() triggers value events that break AntD validators
SEARCH_MOBILE_NTH = 3 on LT and Doctor pages4 search fields exist on those pages (0-indexed), mobile is index 3
Sample capture patternget_by_text("SampleName -", exact=True) then xpath=following::div[1]
Print Bill opens new tabSwitch back with page.context.pages[0].bring_to_front()
Sub-dept navigationBidirectional Next/Prev loop, max 20 attempts

MNC Standard (required on all new code)

Type Hints

All parameters and return values must have type annotations.

Docstrings

All classes and public methods must have docstrings.

BasePage Inheritance

All page classes must inherit from BasePage.

Section Comments

Use # --- Section Name --- to divide logical blocks within methods.
